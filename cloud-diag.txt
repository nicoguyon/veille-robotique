Différence notable par rapport au run 1 : la requête atteint bien l'API Resend cette
fois (401 "API key is invalid", pas de blocage proxy/réseau). Dans le run précédent,
le proxy sortant renvoyait 403 sur le CONNECT vers api.resend.com. Ici l'échec est
uniquement dû à l'absence de RESEND_API_KEY (donc header Authorization invalide) —
la connectivité réseau vers Resend n'est plus bloquée dans cet environnement.
